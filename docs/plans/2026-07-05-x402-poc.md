# x402 PoC Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Wire a $0.01 USDC paywall onto `/api/settlement-risk-report` using x402 v2, prove it works end-to-end on Base Sepolia with an on-chain tx hash.

**Architecture:** `withX402()` from `@x402/next` wraps the route handler, settling payment only after a successful response. A client script uses `wrapFetchWithPaymentFromConfig` from `@x402/fetch` to sign EIP-3009 authorizations automatically. The x402.org free testnet facilitator handles verification (supports `eip155:84532`).

**Tech Stack:** Next.js 16 App Router, `@x402/next` v2.17, `@x402/fetch` v2.17, `@x402/evm` v2.17, viem, Base Sepolia (`eip155:84532`), x402.org facilitator

### Task 2: Create the protected route

**Files:**
- Create: `apps/web/app/api/settlement-risk-report/route.ts`

**Step 1: Write the route handler**

```typescript
// apps/web/app/api/settlement-risk-report/route.ts
import { NextRequest, NextResponse } from "next/server";
import { withX402, x402ResourceServer } from "@x402/next";
import { HTTPFacilitatorClient } from "@x402/core/server";
import { ExactEvmScheme } from "@x402/evm/exact/server";

const facilitator = new HTTPFacilitatorClient({ url: "https://x402.org/facilitator" });
const resourceServer = new x402ResourceServer(facilitator).register(
  "eip155:84532",
  new ExactEvmScheme(),
);

const handler = async (_: NextRequest) => {
  return NextResponse.json({
    generatedAt: new Date().toISOString(),
    counterparty: "Acme Settlement Corp",
    exposureUsd: 142500,
    riskScore: 0.67,
    riskLevel: "MEDIUM",
    factors: ["High leverage", "Concentrated position", "Sector correlation"],
    recommendation: "Reduce exposure by 20% before settlement window.",
  });
};

export const GET = withX402(
  handler,
  {
    accepts: {
      scheme: "exact",
      price: "$0.01",
      network: "eip155:84532",
      payTo: "0xe9F97E2F7c6DCB8FCdBCDFBA074334D22a6c3117",
    },
    description: "Settlement risk report — $0.01 per request",
  },
  resourceServer,
);
```

### Task 4: Write the client script

**Files:**
- Create: `scripts/pay-and-fetch.ts`

**Step 1: Write the script**

```typescript
// scripts/pay-and-fetch.ts
import { wrapFetchWithPaymentFromConfig, decodePaymentResponseHeader } from "@x402/fetch";
import { ExactEvmScheme } from "@x402/evm";
import { privateKeyToAccount } from "viem/accounts";

const privateKey = process.env.TESTNET_PRIVATE_KEY;
if (!privateKey) throw new Error("TESTNET_PRIVATE_KEY not set in env");

const account = privateKeyToAccount(privateKey as `0x${string}`);

const fetchWithPayment = wrapFetchWithPaymentFromConfig(fetch, {
  schemes: [
    {
      network: "eip155:84532",
      client: new ExactEvmScheme(account),
    },
  ],
});

console.log("Calling protected route...");
const response = await fetchWithPayment("http://localhost:3000/api/settlement-risk-report");

const paymentHeader = response.headers.get("PAYMENT-RESPONSE");
if (paymentHeader) {
  const decoded = decodePaymentResponseHeader(paymentHeader);
  console.log("Tx hash:", decoded.transaction);
  console.log("Basescan:", `https://sepolia.basescan.org/tx/${decoded.transaction}`);
}

const data = await response.json();
console.log("Status:", response.status);
console.log("Report:", JSON.stringify(data, null, 2));
```

### Task 5: Run the end-to-end flow

Make sure the dev server from Task 3 is still running on port 3000.

**Step 1: Run the client script**

```bash
bun run scripts/pay-and-fetch.ts
```

Expected flow:
1. Script makes GET to `/api/settlement-risk-report`
2. Server returns 402 with payment requirements
3. `@x402/fetch` signs the EIP-3009 authorization using the private key
4. Script retries with `X-PAYMENT` header
5. x402.org facilitator verifies and settles on-chain
6. Server returns 200 with the report JSON
7. `PAYMENT-RESPONSE` header contains the tx hash

Expected output:
```
Calling protected route...
Tx hash: 0x<64-char hex>
Basescan: https://sepolia.basescan.org/tx/0x<64-char hex>
Status: 200
Report: {
  "generatedAt": "...",
  "counterparty": "Acme Settlement Corp",
  ...
}
```

If you get a 402 back from the retry (not the initial request), the payment was rejected by the facilitator — likely insufficient balance. Run `bun run scripts/check-balance.ts` to verify USDC balance is > 0.

If you get an error about `ExactEvmScheme` or missing module, ensure `@x402/evm` is installed at root: `bun add @x402/evm`.
