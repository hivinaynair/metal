# x402 Payment Protocol PoC Design

## Context

Proof-of-concept integrating x402 v2 into the bare-metal Next.js monorepo. Goal: validate the full payment round-trip on Base Sepolia — 402 → EIP-3009 sign → 200 with data → on-chain tx confirmed.

## Decision: Facilitator

Use the **x402.org free testnet facilitator** (`https://x402.org/facilitator`). It explicitly supports Base Sepolia (`eip155:84532`). The Coinbase CDP facilitator is mainnet-only.

## Architecture

### Protected Route

- **File**: `apps/web/app/api/settlement-risk-report/route.ts`
- **Pattern**: Route handler wrapper (`withX402()` from `@x402/next`) — charges only after a successful response, correct semantics for data-bearing routes
- **Payment config**:
  - `payTo`: `0xe9F97E2F7c6DCB8FCdBCDFBA074334D22a6c3117`
  - `price`: `$0.01` USDC
  - `network`: `eip155:84532`
  - `facilitator`: `https://x402.org/facilitator`
- **Response**: Stub JSON with realistic settlement risk report fields

### Client Script

- **File**: `scripts/pay-and-fetch.ts`
- **Library**: `@x402/fetch` wraps native fetch with automatic payment handling
- **Wallet**: Loads `TESTNET_PRIVATE_KEY` from env, creates viem wallet client on Base Sepolia
- **Flow**: Calls route → receives 402 → signs EIP-3009 authorization → resubmits → logs 200 body + tx hash

### Verification

- Extract tx hash from `X-Payment-Response` header or response body
- Confirm on `https://sepolia.basescan.org/tx/<hash>`

## Packages

- `@x402/next` — route handler wrapper
- `@x402/fetch` — client payment helper
- Both installed in `apps/web` and root respectively
